# FlowAgent 产品说明

> 最后更新：2026-04-02

## 一、FlowAgent 是什么

FlowAgent 是一个 **AI 驱动的业务翻译平台**。

它解决的核心问题是：**业务人员有想法但说不清楚，技术人员能做但不知道做什么。**

传统做法是业务方写需求文档 → 产品经理翻译 → 技术方评估 → 反复沟通。这个过程通常需要 1-2 周，而且信息在每次传递中都会失真。

FlowAgent 让业务方用自然语言描述业务场景，AI 自动翻译成结构化的可执行方案，技术方在同一个平台上直接评审。**从"描述"到"方案确认"，可以在一次对话中完成。**

---

## 二、核心能力

### 2.1 AI 业务翻译

业务方在输入框中描述场景（比如"我想自动化进出口报关流程"），AI 会：

1. **判断任务类型** — 自动识别这是一个有固定步骤的"工作流"，还是一个需要 AI 自主规划的"智能体"任务
2. **生成结构化方案** — 工作流生成可视化流程图，智能体生成任务配置（目标/技能/约束/评估体系）
3. **逐项确认** — 对不确定的部分逐个询问业务方，确保方案准确

### 2.2 双角色协作

平台区分两种角色：

| 角色 | 做什么 | 看到什么 |
|------|--------|----------|
| **业务方** | 描述需求、编辑方案、提交评审 | 流程图/配置编辑器、知识中心（业务文档） |
| **技术方** | 评审可行性、添加批注、通过/打回 | 方案只读视图、批注面板、知识中心（业务文档 + 技术参考） |

两种角色通过不同入口进入，在同一个方案上协作，避免信息在传递中失真。

### 2.3 两种任务类型

| 类型 | 适用场景 | AI 输出 | 示例 |
|------|---------|---------|------|
| **工作流（Workflow）** | 有明确步骤顺序的流程 | 可视化流程图（节点 + 连线） | 进出口报关、财务报销审批 |
| **智能体（Agentic）** | 目标导向、需要 AI 自主规划的任务 | 任务配置（目标/技能/约束/评估器） | 小红书账号运营、竞品分析 |

AI 会自动判断类型，业务方不需要理解技术概念。

### 2.4 管控后台

方案确认后，可以提交到管控后台。管控后台提供：

- **运营仪表盘** — 一眼看到有多少 Agent 在运行、任务成功率、待处理事项
- **任务监控** — 每个任务的执行进度、当前节点、耗时
- **人工介入** — 需要人工确认的节点会暂停等待，异常任务可以重试/接管/跳过

## 三、当前状态

### 已完成的功能

| 模块 | 状态 | 说明 |
|------|------|------|
| AI 业务翻译（Workflow） | ✅ 可用 | 输入描述 → AI 分类 → 生成流程图 → 逐节点确认 → 自由修改 |
| AI 业务翻译（Agentic） | ✅ 可用 | 输入描述 → AI 分类 → 生成配置 → 逐模块确认 → 自由修改 |
| 流程图编辑器 | ✅ 可用 | 拖拽、添加/删除节点、编辑节点详情、画布缩放 |
| Agentic 配置面板 | ✅ 可用 | 4 个 Tab（目标/技能/约束/评估器），支持编辑、技能市场集成 |
| 双角色体系 | ✅ 可用 | 业务方/技术方独立入口，编辑器内角色标签 + 切换 |
| 技术方评审 | ✅ 可用 | 4 个 Mock 场景直接加载，批注/回复、评审通过/打回 |
| 批注系统 | ✅ 可用 | 按节点批注（Workflow）/ 全局批注（Agentic），回复、标记已解决 |
| 知识面板 | ✅ 可用 | 业务文档 + 技术参考（技术方双 Tab） |
| 提交至管控后台 | ✅ 可用 | 生成配置 JSON 摘要，可复制 |
| 管控后台仪表盘 | ✅ 可用 | 统计卡片、待处理事项、最近任务、Agent 概览 |
| Agent 团队管理 | ✅ 可用 | Agent 卡片列表（状态/成功率/任务数） |
| 任务监控 | ✅ 可用 | 筛选表格 + 任务详情页（时间线 + 人工确认/异常处理） |

### 当前限制

| 限制 | 说明 |
|------|------|
| 无真实后端 | 管控后台的数据全部是 Mock，没有真实的 Agent 运行时 |
| 无用户系统 | 角色通过 URL 区分，没有登录/注册 |
| 无数据持久化 | 数据存在浏览器 localStorage，刷新可能丢失 |
| 技术方评审为 Mock | 4 个评审场景是预设数据，不是业务方真实提交的 |
| 无通知集成 | 没有钉钉/飞书通知能力 |

## 四、用户使用路径

### 业务方路径

```
1. 打开首页 (/)
2. 在输入框描述业务场景（或点击示例卡片）
3. AI 自动判断类型，生成方案草稿
4. 逐项确认 AI 不确定的部分
5. 在编辑器中查看和修改方案
6. 满意后点击「提交至管控后台」
7. 查看生成的配置 JSON 摘要
```

### 技术方路径

```
1. 打开技术方入口 (/tech)
2. 查看待评审列表（统计卡片 + 评审项）
3. 点击某个评审项，进入编辑器
4. 查看业务方提交的方案（流程图或 Agentic 配置）
5. 打开批注面板，添加技术评审意见
6. 打开知识面板，查看业务文档和技术参考
7. 点击「评审通过」或「打回修改」
```

## 五、产品价值

### 对业务方

- **降低表达门槛** — 不需要写 PRD，用自然语言描述即可
- **即时可视化** — 说完就能看到流程图/配置方案，不用等产品经理翻译
- **减少沟通成本** — 在同一个平台上和技术方协作，不用反复开会

### 对技术方

- **结构化输入** — 拿到的不是模糊需求，而是有明确步骤、输入输出、约束条件的结构化方案
- **评审有据可依** — 每个节点的执行方式、预估耗时、错误处理都已定义
- **批注留痕** — 评审意见直接挂在方案上，不会丢失

### 对组织

- **缩短需求确认周期** — 从"周级"缩短到"小时级"
- **减少信息失真** — 业务方的原始描述和 AI 翻译结果在同一个界面
- **积累组织知识** — 每次翻译的结果都是可复用的流程模板

## 六、后续优化计划

### Phase 1：补齐核心体验（1-2 周）

| 优先级 | 事项 | 目标 |
|--------|------|------|
| P0 | **用户系统** | 基础登录/注册，角色绑定到账号而非 URL |
| P0 | **数据持久化** | 方案保存到服务端数据库，支持历史版本 |
| P0 | **业务方提交 → 技术方收到** | 打通真实的提交-评审流程，替代当前 Mock |
| P1 | **通知集成** | 业务方提交后通过飞书/钉钉通知技术方 |
| P1 | **方案版本管理** | 支持查看修改历史、对比不同版本 |
| P1 | **AI 翻译质量优化** | 优化 prompt 工程，提升流程图/配置的准确度 |

### Phase 2：打通执行闭环（2-4 周）

| 优先级 | 事项 | 目标 |
|--------|------|------|
| P0 | **Agent 运行时对接** | 对接 LangGraph / Dify 等平台，实现真正的一键部署 |
| P0 | **管控后台真实数据** | 从 Agent 运行时拉取真实的任务状态和执行日志 |
| P1 | **人工确认节点** | Agent 执行到需要人工确认的节点时，通过通知推送给相关人 |
| P1 | **异常告警** | Agent 执行异常时自动告警，支持重试/人工接管 |
| P2 | **执行数据看板** | 基于真实数据的成功率、耗时、异常率统计 |

### Phase 3：规模化与生态（1-3 月）

| 优先级 | 事项 | 目标 |
|--------|------|------|
| P1 | **技能市场** | 从 Mock 升级为真实的 Skill 注册/发现/调用平台 |
| P1 | **场景模板库** | 将已确认的方案沉淀为可复用的场景模板 |
| P2 | **混合任务类型** | 支持一个方案中同时包含 Workflow 和 Agentic 节点 |
| P2 | **多租户** | 支持不同团队/部门独立使用，数据隔离 |
| P2 | **权限管理** | RBAC 权限模型，细粒度控制不同角色的操作范围 |
| P3 | **审计日志** | 完整的操作审计记录 |
| P3 | **数据驾驶舱** | 业务目标达成率、人工介入率、异常率趋势等高级分析 |

### 技术债务清理

| 事项 | 说明 |
|------|------|
| 编辑器 ref 逻辑重构 | 当前用多个 `useRef` 做初始化守卫，跨场景导航可能失效，需改为 key-based 重挂载 |
| Store 持久化策略优化 | `partialize` 遗漏部分状态，`loadGeneratedFlow` 会覆盖 project.status |
| 响应式适配 | 首页特性标签在窄屏溢出，编辑器在小屏幕下布局需优化 |
| 错误边界 | API 调用失败、JSON 解析失败等场景缺少用户友好的错误提示 |
